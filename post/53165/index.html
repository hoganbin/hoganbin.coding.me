基于hexo搭建博客的心得过程

发表于 2018-04-17   |  阅读量 次 | 分类于 博客搭建 | |
字数统计: 821 | 阅读时长 ≈ 2

在搭建博客过程中，确实让我学到了很多以前不知道的前端开发知识，虽然我有看到很多高中生都有了自己的博客，可能他们是全国高中生信息竞赛的学生，这我不觉得足以为奇，或许是因为他们的兴趣爱好。

目前搭建博客最受欢迎又是免费的还就是hexo与github，首先前提工作是安装3个必备的软件，分别sublime（网页制作编辑器），git（代码托管仓库），node.js（服务器端 JavaScript 解释器,）；然后就是注册coding账号，它提供了page服务，是我们用来放在网站文件的免费“服务器”。注意到：同类的代码托管网站有gitee、github等。

这次我选择国内速度比较快，功能完善的Coding ，其他托管网站也可使用，与Coding 大同小异；再然后就是安装hexo ，在电脑桌面创建blog文件夹，再切换NPM源，这里利用git bash输入命令行代码npm install -g cnpm –registry，再输入cnpm install -g hexo-cli，使用hexo -v 指令查看hexo的版本，使用hexo init指令初始化hexo，使用hexo s 指令预览网站，这时候我们选择允许访问hexo就已经运行在这个网址 ，我们可使用Ctrl+C组合键来停止预览。然后就可看到在我们的Blog文件夹出现如下的目录结构，意思都注释在右边。

继续下一步，就是配置SSH部署公钥，还有就是配置博客网站，打开我们建立的Blog文件夹，如下图我们使用Sublime打开_config.yml文件，对博客进行配置，就出现如下图，我们可以随意更改的有tittle网站标题，subtitle小标题， description网站描述。配置文件拉到最下面 (重要)，本部分修改的是

上传的地址，是网站是否能成功上传到服务器的关键。

然后就是配置Git和Git部署插件，使用命令git config –global user.email “huzhibin2014sian@163.com“配置我的Git邮箱，使用命令 git config –global user.name “hoganbin”配置我的Git用户名。然后再使用使用cnpm install hexo-deployer-git –save来安装git上传插件。

就hexo -g渲染本地文件，hexo -d上传到网站仓库，就可以成功访问自己的域名网站了，此时你的博客就成功搭建成功了，开不开心！整个教程实践操作只需要两个小时，后面对主题配置有33种之多，需要花时间。具体hexo的next主题个性化教程的33种配置如下链接就可以操作。hexo的next主题个性化配置

昨晚我又对博客又改变了一些东西，看下面，哈哈。增加了友情链接，有好几个是我的朋友博客，与他们互友，包括主页的背景音乐，包括自己常浏览的网站与自己拥有的论坛账号，在分类栏中，我将自己学习分类，这样在将来的某一天回过去看曾经的自己写了什么，可以很好的记录下来。

就这样，整个博客基本操作都完成了。
